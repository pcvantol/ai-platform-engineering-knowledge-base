# Prompt Architecture

This area defines the conceptual role of prompts for future AI-assisted CLI behavior.

Prompts are support assets. Prompts are not canonical knowledge. Prompts must reference Certified Knowledge rather than duplicate it.

This area does not create operational prompts or implement AI workflows.

## Prompt Responsibilities

Prompts may guide AI assistance for:

- repository analysis;
- observation discovery;
- candidate drafting;
- relationship suggestions;
- classification suggestions;
- review packet preparation;
- publication drafting;
- engineering question answering;
- quality and health reporting.

Prompts must not:

- certify knowledge;
- modify Certified Knowledge;
- replace governance;
- invent evidence;
- embed canonical engineering conclusions without references;
- treat publication text as canonical.

## Prompt Inputs

Future prompts should receive:

- command context;
- Certified Knowledge references;
- relevant relationships;
- source evidence references;
- required output format;
- uncertainty and traceability requirements;
- governance boundary reminders.

## Prompt Outputs

Prompt outputs should remain:

- explainable;
- traceable;
- reviewable;
- explicitly non-authoritative;
- separated from evidence.

AI-generated prompt output becomes actionable only through governed review or command orchestration.
