# Improvement Report Examples

This area documents example uses of `aikb improve`.

Improvement Reports are proposal artifacts. They do not modify the Knowledge Base and they do not approve implementation.

## Example Commands

```text
bin/aikb improve
```

```text
bin/aikb improve knowledge
```

```text
bin/aikb improve generators
```

```text
bin/aikb improve publications
```

```text
bin/aikb improve generators --output /tmp/aikb-generator-improvement-report.md
```

## Report Contents

Each improvement report contains:

- repository metrics;
- validation evidence;
- improvement analysis;
- objective metrics;
- supporting evidence;
- engineering rationale;
- estimated effort;
- engineering value;
- governance path;
- improvement recommendations.

## Read-Only Boundary

`aikb improve` does not:

- modify Certified Knowledge;
- modify repositories;
- change governance;
- approve improvements;
- rewrite engineering history;
- rewrite publications;
- rewrite generators or templates.
