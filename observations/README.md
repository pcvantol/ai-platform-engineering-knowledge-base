# Repository Observations

This area contains repository observations, also referred to as Engineering Observations in the extraction framework.

A repository observation is captured evidence from a software repository or engineering system. Observations may originate from architecture decisions, implementation experience, verification evidence, design discussions, experiments, failures, production incidents, operational experience, or successful engineering practices.

Repository observations are not knowledge. They are evidence that may later support knowledge candidates.

Every observation must preserve enough source context to remain traceable even when later lifecycle stages generalize or certify the knowledge derived from it.

The canonical extraction rules for creating observations are defined in [../extraction/README.md](../extraction/README.md).
