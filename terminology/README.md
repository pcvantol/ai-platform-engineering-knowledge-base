# Terminology

This area contains the canonical terminology used by the Knowledge Base.

Terminology ensures that lifecycle stages, classifications, governance language, and publication language are used consistently across the repository.

Terms defined here must remain project independent and technology independent. Product-specific vocabulary may be referenced only when it is evidence for a reusable term.

Canonical lifecycle terms are defined in [../lifecycle/README.md](../lifecycle/README.md). This area may later contain a more detailed controlled vocabulary, but it must not conflict with the lifecycle terminology.

Canonical model terms are defined in [../model/README.md](../model/README.md). Future terminology entries must align with the Knowledge Object, metadata, relationship, abstraction, certification, and versioning definitions.

Canonical classification terms are defined in [../classification/README.md](../classification/README.md). Classification terminology must remain independent from lifecycle terminology.

Canonical query and retrieval terms are defined in [../query-retrieval/README.md](../query-retrieval/README.md). Retrieval terminology must preserve the distinction between read-only discovery, AI-generated explanations, and canonical knowledge.

Canonical synthesis terms are defined in [../synthesis/README.md](../synthesis/README.md). Synthesis terminology must preserve the distinction between derived understanding, source knowledge, and Certified Knowledge.

Canonical automation terms are defined in [../automation/README.md](../automation/README.md). Automation terminology must preserve the distinction between operational support, automated proposals, and governed decisions.

Canonical source onboarding terms are defined in [../source-onboarding/README.md](../source-onboarding/README.md). Onboarding terminology must preserve the distinction between proposed repositories, approved Knowledge Sources, profiles, and extraction boundaries.

Canonical extraction terms are defined in [../extraction/README.md](../extraction/README.md). Extraction terminology must preserve the distinction between evidence, observations, candidates, and certified knowledge.

Canonical certification terms are defined in [../certification/README.md](../certification/README.md). Certification terminology must remain separate from lifecycle terminology.

Canonical publication terms are defined in [../publications/README.md](../publications/README.md). Publication terminology must preserve the distinction between derived artifacts and canonical knowledge.

Canonical AI operations terms are defined in [../ai-operations/README.md](../ai-operations/README.md). AI terminology must preserve the distinction between assistance, proposals, and engineering authority.

Canonical source integration terms are defined in [../sources/README.md](../sources/README.md). Source terminology must preserve the distinction between external evidence sources and the canonical Knowledge Base.

Canonical ingestion terms are defined in [../ingestion/README.md](../ingestion/README.md). Ingestion terminology must preserve the distinction between observations, candidate proposals, registered candidates, and certified knowledge.

Canonical review and promotion terms are defined in [../review-promotion/README.md](../review-promotion/README.md). Promotion terminology must preserve the distinction between review recommendations, promotion decisions, and certification.
