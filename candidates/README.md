# Knowledge Candidates

This area contains knowledge candidates.

A knowledge candidate is an initial knowledge item derived from repository observations. A candidate is not canonical.

Candidates must remain traceable to evidence and must not assume that the originating project is authoritative.

The canonical pipeline for proposing and registering candidates is defined in [../ingestion/README.md](../ingestion/README.md).

The operational candidate generation command is `aikb classify`, documented in [../cli/README.md](../cli/README.md). It transforms Engineering Observations into Knowledge Candidates only.
